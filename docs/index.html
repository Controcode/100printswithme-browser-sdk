100PrintsWithMe Browser SDK

Installation #

Install the SDK from npm. All rendering dependencies are bundled — no peer dependencies are required.

npm npm install @100printswithme/browser-sdk
pnpm pnpm add @100printswithme/browser-sdk
yarn yarn add @100printswithme/browser-sdk

CDN (UMD)

Include the UMD build directly in any HTML page without a build step.

HTML 
<script src="https://cdn.jsdelivr.net/npm/@100printswithme/browser-sdk/dist/100prints-sdk.umd.js"></script>
<script>
  const sdk = new BrowserSDK100Prints.BrowserSDK({
    key: 'pk_live_...'
  });
</script>

Quick Start #

Initialize the SDK with your publishable key and render a template to a downloadable PDF in under 10 lines of code.

TypeScript
import { BrowserSDK } from '@100printswithme/browser-sdk';

// 1. Initialize once (use your publishable key — safe to expose)
const sdk = new BrowserSDK({
  key: 'pk_live_xxxxxxxxxxxxxxxxxx',
});

// 2. Render a template to PDF
const result = await sdk.render({
  templateId: 'template_abc123',
  payload: {
    name:  'Rishabh Dev',
    email: 'rishabh@example.com',
    role:  'Software Engineer',
    photo: 'https://cdn.example.com/photo.jpg',
  },
  format:  'pdf',
  quality: 'high',
  side:    'both',
});

// 3. Trigger browser download
const url  = URL.createObjectURL(result.blob);
const link = document.createElement('a');
link.href     = url;
link.download = 'badge.pdf';
link.click();
URL.revokeObjectURL(url);
JavaScript
import { BrowserSDK } from '@100printswithme/browser-sdk';

const sdk = new BrowserSDK({ key: 'pk_live_...' });

const { blob } = await sdk.render({
  templateId: 'template_abc123',
  payload: { name: 'Rishabh Dev', role: 'Engineer' },
  format: 'png',
});

const url = URL.createObjectURL(blob);
document.getElementById('preview-img').src = url;
React
import { useRef, useCallback } from 'react';
import { BrowserSDK } from '@100printswithme/browser-sdk';

// Initialize once outside the component (stable across renders)
const sdk = new BrowserSDK({ key: 'pk_live_...' });

export function BadgeExporter({ employee }) {
  const previewRef = useRef(null);

  const handlePreview = useCallback(async () => {
    await sdk.preview({
      templateId: 'template_abc123',
      payload:    employee,
      container:  previewRef.current,
    });
  }, [employee]);

  const handleExport = useCallback(async () => {
    const { blob } = await sdk.render({
      templateId: 'template_abc123',
      payload: employee,
      format: 'pdf', quality: 'high',
    });
    const a = document.createElement('a');
    a.href     = URL.createObjectURL(blob);
    a.download = 'badge.pdf';
    a.click();
  }, [employee]);

  return (
    <div>
      <div ref={previewRef} style={{ width: 428, height: 270 }} />
      <button onClick={handlePreview}>Preview</button>
      <button onClick={handleExport}>Export PDF</button>
    </div>
  );
}

Authentication #

The SDK uses a publishable key — it is safe to include in client-side JavaScript. Publishable keys have the prefix pk_live_. Never use your secret API key on the client side.

ℹ

Publishable keys are scoped to specific templates and plans. You control which templates a key may access from the 100Prints dashboard.

Key Permissions

Each key is granted access via the keys table which stores fine-grained permissions per key:

Field Description
key Publishable key string — safe to use in browser code
key_type Always sdk for Browser SDK keys. Other types include wordpress etc.
permission JSON object. { "templates": "all" } or { "templates": ["id1", "id2"] }
plan_required Endpoint enforces non-free plan (Pro, Enterprise, etc.)
⚠

SDK access requires at least a Pro plan. Free accounts cannot generate publishable keys.

BrowserSDK() #

Creates an SDK instance. Instantiate once and reuse across your application — the SDK internally caches template data.

new BrowserSDK(options: BrowserSDKOptions): BrowserSDK
ParameterTypeRequiredDescription
key string required Your publishable key (pk_live_...)
baseUrl string optional Override the API base URL. Defaults to https://api.100printswith.me
TypeScript
import { BrowserSDK } from '@100printswithme/browser-sdk';

const sdk = new BrowserSDK({
  key: 'pk_live_xxxxxxxxxxxxxxxxxx',
  // Optional: override API base for self-hosted deployments
  baseUrl: 'https://my.100prints.instance.com',
});

sdk.render() #

Fetches a template from the API, resolves all variable placeholders with the provided payload, and renders it to a Blob — either as a PDF or PNG. Template data is cached after the first fetch.

sdk.render(options: RenderOptions): Promise<RenderResult>

Parameters

ParameterTypeRequiredDescription
templateId string required The template identifier from 100Prints
payload Record<string, any> optional Key-value pairs that replace {{variable}} placeholders in the template
format 'pdf' | 'png' optional Output format. Defaults to 'pdf'
quality 'draft' | 'standard' | 'high' | 'ultra' optional Export DPI. draft=72, standard=150, high=300, ultra=600. Defaults to 'high'
side 'front' | 'back' | 'both' optional Which side(s) to render. PDF renders both as pages. PNG renders front unless 'back' is specified. Defaults to 'both'

Returns — RenderResult

FieldTypeDescription
blobBlobThe rendered file as a browser Blob
mimeTypestringapplication/pdf or image/png
sizeKBnumberApproximate output file size in kilobytes

Quality / DPI Reference

ValueDPIScale FactorBest For
draft721×Screen preview, rapid generation
standard1502×Digital-only distribution
high3004×Print-ready — recommended
ultra6008×Large-format / high-end print
TypeScript — PDF export with full options
const result = await sdk.render({
  templateId: 'template_abc123',
  payload: {
    name:       'Rishabh Dev',
    email:      'rishabh@example.com',
    role:       'Senior Engineer',
    department: 'Platform',
    photo:      'https://cdn.example.com/photo.jpg',
    qr_data:    'https://example.com/badge/12345',
  },
  format:  'pdf',   // print-quality PDF
  quality: 'high',  // 300 DPI
  side:    'both',  // front + back as separate pages
});

// result.blob     → Blob (application/pdf)
// result.mimeType → 'application/pdf'
// result.sizeKB   → e.g. 284

downloadBlob(result.blob, 'badge.pdf');

// Utility helper:
function downloadBlob(blob: Blob, filename: string) {
  const a = document.createElement('a');
  a.href = URL.createObjectURL(blob);
  a.download = filename;
  a.click();
  URL.revokeObjectURL(a.href);
}

sdk.preview() #

Renders a live interactive Konva canvas directly into a DOM container. Ideal for showing a real-time preview to users before they confirm an export. No blob is generated — this is rendering-only.

sdk.preview(options: PreviewOptions): Promise<HTMLCanvasElement>

Parameters

ParameterTypeRequiredDescription
templateId string required The template to preview
container HTMLElement required DOM element into which the Konva canvas is rendered. Existing children are cleared.
payload Record<string, any> optional Variable values to substitute. Pass an empty object {} to see default values.
scale number optional Scales the Konva stage. 0.5 renders at half size, 1 (default) renders at native template dimensions.
✓

The container element is sized by the Konva stage, not by your CSS. Set a max-width wrapper and use scale to control the canvas size in the page.

TypeScript
const container = document.getElementById('preview-area')!;

// Render front at 50% scale (fits in a 428px wide container)
const canvas = await sdk.preview({
  templateId: 'template_abc123',
  container:  container,
  payload:    { name: 'Rishabh Dev', role: 'Engineer' },
  scale:      0.5,
});

// Returns the raw HTMLCanvasElement if you need further operations
canvas.toBlob((blob) => { /* custom usage */ });

sdk.renderBulk() #

Renders multiple records against a single template and packages the results as either a merged PDF (one page per record) or a ZIP archive containing individual PNG or PDF files.

sdk.renderBulk(options: BulkRenderOptions): Promise<BulkRenderResult>

Parameters

ParameterTypeRequiredDescription
templateId string required Template to render for all rows
rows Record<string, any>[] required Array of payload objects. One document is generated per row.
format 'pdf' | 'png' optional Format for individual records. Defaults to 'pdf'
quality 'draft' | 'standard' | 'high' | 'ultra' optional Export DPI. Defaults to 'high'
mode 'merged' | 'zip' optional 'merged': single PDF with all records as pages (PDF only). 'zip': individual files in a ZIP archive. Defaults to 'merged'
onProgress (current, total, name) => void optional Progress callback called after each record is rendered. Use to update a progress bar.

Returns — BulkRenderResult

FieldTypeDescription
blobBlobThe output file (merged PDF or ZIP archive)
filenamestringSuggested filename for the download, e.g. BadgeTemplate_Export.pdf
sizeKBnumberOutput file size in kilobytes
TypeScript — Bulk export with progress
const employees = [
  { name: 'Rishabh Dev',  role: 'Engineer',   dept: 'Platform' },
  { name: 'Priya Sharma', role: 'Designer',   dept: 'Product'  },
  { name: 'Alex Johnson', role: 'Manager',    dept: 'Ops'      },
];

const result = await sdk.renderBulk({
  templateId: 'template_abc123',
  rows:        employees,
  format:      'pdf',
  quality:     'high',
  mode:        'merged',
  onProgress:  (current, total, name) => {
    progressEl.textContent = `Rendering ${name} (${current}/${total})`;
    barEl.style.width       = `${(current / total) * 100}%`;
  },
});

// result.blob     → merged 3-page PDF Blob
// result.filename → 'BadgeTemplate_Export.pdf'
// result.sizeKB   → e.g. 812

downloadBlob(result.blob, result.filename);

sdk.destroy() #

Clears the internal template cache. Call this when unmounting the component that owns the SDK instance, or in single-page app route teardown.

sdk.destroy(): void
React — cleanup in useEffect
import { useEffect, useRef } from 'react';
import { BrowserSDK } from '@100printswithme/browser-sdk';

export function MyComponent() {
  const sdkRef = useRef<BrowserSDK>(null);

  useEffect(() => {
    sdkRef.current = new BrowserSDK({ key: 'pk_live_...' });
    return () => { sdkRef.current?.destroy(); };
  }, []);

  // ...
}

Guide: Building Payloads #

Templates use {{variableName}} mustache syntax. The payload object you pass to render(), preview(), or each row in renderBulk() must contain keys matching those variable names.

ℹ

Variable names are case-sensitive. {{Name}} and {{name}} are different variables.

Image Variables

Pass a publicly accessible URL for image layers. The SDK will preload the image before rendering to avoid async artifacts.

TypeScript — payload with images
await sdk.render({
  templateId: 'template_abc123',
  payload: {
    // Text variables
    name:       'Rishabh Dev',
    title:      'Senior Software Engineer',
    department: 'Platform Engineering',
    employee_id:'EMP-4921',

    // Image variable — must be a public URL
    photo:      'https://cdn.example.com/avatars/rishabh.jpg',

    // QR code / barcode — pass the encoded data string
    qr_data:    'https://example.com/employee/4921',
  },
  format: 'pdf',
});

Missing Variables

If a variable in the template is not present in the payload, it renders as an empty string. No error is thrown.

Guide: Bulk Rendering #

Bulk rendering loops over an array of row objects and renders each one against the same template. Use mode: 'merged' for a single PDF export, or mode: 'zip' for individual files.

Merged PDF (one PDF, multiple pages)

TypeScript
const result = await sdk.renderBulk({
  templateId: 'template_abc123',
  rows:    employees,
  format:  'pdf',
  quality: 'high',
  mode:    'merged',  // → TemplateName_Export.pdf (N pages)
});

ZIP Archive (one file per record)

TypeScript
const result = await sdk.renderBulk({
  templateId: 'template_abc123',
  rows:    employees,
  format:  'png',
  quality: 'standard',
  mode:    'zip',      // → TemplateName_Export.zip
                        //   contains: Rishabh Dev_Front.png, etc.
  onProgress: (cur, tot, name) => {
    console.log(`[${cur}/${tot}] Rendered: ${name}`);
  }
});
⚠

Bulk rendering runs entirely in the browser. For very large batches (500+ records at ultra quality), consider chunking the operation to avoid tab crashes on lower-end devices. Use 'draft' or 'standard' quality for initial proofing.

Guide: Framework Integration #

React Hook
// hooks/use100Prints.ts
import { useMemo } from 'react';
import { BrowserSDK } from '@100printswithme/browser-sdk';

const _sdk = new BrowserSDK({ key: process.env.NEXT_PUBLIC_100PRINTS_KEY! });

export function use100Prints() {
  return useMemo(() => _sdk, []);
}

// Usage in a component:
export function ExportButton({ id, data }) {
  const sdk = use100Prints();
  const handleExport = async () => {
    const { blob, mimeType } = await sdk.render({
      templateId: id, payload: data, format: 'pdf'
    });
    const url = URL.createObjectURL(new Blob([blob], { type: mimeType }));
    window.open(url, '_blank');
  };
  return <button onClick={handleExport}>Export PDF</button>;
}

Errors & Status Codes #

The SDK throws a plain Error when the API returns a non-OK response. The error message contains the HTTP status and a description from the server.

TypeScript — error handling
try {
  const result = await sdk.render({ templateId: '...', payload: {} });
} catch (err) {
  if (err instanceof Error) {
    console.error(err.message);
    // e.g. "API Error: 403 Forbidden — Template not accessible with this key"
  }
}

Common Error Scenarios

  • 401 Unauthorized — The API key is missing, invalid, or has been revoked. Verify pk_live_... key is correct.
  • 402 Payment Required — SDK access requires a Pro or Enterprise plan. Upgrade from the 100Prints dashboard.
  • 403 Forbidden — The key exists but does not have permission for the requested template. Check key permissions in the dashboard.
  • 404 Not Found — No template exists with the provided templateId. Check the ID in your 100Prints dashboard.
  • 429 Too Many Requests — Rate limit exceeded. The SDK API allows 5 requests per minute per key. Implement exponential backoff.
✓

For bulk rendering, only the initial template fetch counts against the rate limit. Subsequent rows re-use the cached template and render entirely in the browser.

Rate Limits #

Rate limits apply to the /public/v1/sdk/render API endpoint — which is called once per unique templateId to fetch template data. The rendering itself is fully client-side.

LimitValueScope
Requests per minute 5 req/min Per publishable key
Template cache In-memory SDK instance (cleared on destroy())
Bulk rendering Unlimited Client-side only, no API calls after template fetch

Create multiple SDK instances with different keys if you need higher throughput, or contact support for enterprise rate limit increases.
